# Rudder Control-Plane — Phase 2 Design

_GitOps Control Plane backend · 2026-06-02_

## Goal

Make the GitHub/Azure DevOps/Vault integrations **actually work** end-to-end: a
containerized control-plane that pulls a Git repo, renders its job manifest into
a schedule, runs real Ansible jobs against hosts (auth via Vault), emits
metrics + logs, and exposes a REST API the web-UI binds to live (no more mock
data). Tested fully on local Docker.

## Locked decisions

| Decision | Choice |
|---|---|
| Git source for the test | **Bundled Gitea**, seeded with a sample `fleet` repo (self-contained, no external secrets) |
| Run target | **Bundled SSH target container**; real `ansible-playbook` over SSH using a key from Vault |
| Stack | **Python 3.12 + FastAPI + APScheduler + Ansible + hvac + PyYAML + prometheus_client** |
| Providers | git/Gitea, GitHub, and Azure DevOps adapters (clone URL + token auth); test uses Gitea |
| UI data | Rewired mock → **live API** via a `useData()` provider; "add repo" → `POST /repos` |
| Secrets | Nothing in git. Vault SSH key generated at runtime; Gitea token generated at runtime (shared volume); creds via gitignored `.env` |

## Components (containers, segmented)

- **gitea** — local Git server. `gitea-init` (one-shot) creates an admin + access
  token (written to a shared volume). control-plane seeds the `rudder/fleet` repo
  via the Gitea API on first boot.
- **control-plane** (new) — reconcile loop + scheduler + Ansible runner + Vault +
  telemetry + REST API.
- **target** — Linux + sshd + python3; entrypoint pulls the authorized public key
  from Vault, then runs sshd. The Ansible run target.
- **vault** (OpenBao) — holds the SSH keypair (generated by control-plane on boot)
  + demo secret references.
- **prometheus + pushgateway**, **loki** — telemetry sinks.
- **web-ui** — live mode binds to `/api/control-plane/*`.

## Control-plane REST API (served at root; reached via nginx `/api/control-plane/*`)

| Method | Path | Returns |
|---|---|---|
| GET | `/healthz` | `{status:"ok"}` |
| GET | `/repos` | `ConnectedRepo[]` |
| POST | `/repos` | add `{provider,url,branch}` → clone + reconcile |
| DELETE | `/repos/{id}` | remove |
| GET | `/reconcile` | `{lastAt,intervalMin,inSync,nextAt}` |
| POST | `/reconcile` | force pull + re-render |
| GET | `/jobs` | `Job[]` (rendered from manifests + latest run status) |
| GET | `/jobs/{name}` | `Job` with `runs[]` |
| POST | `/jobs/{name}/run` | trigger a run now |
| GET | `/activity` | `ActivityItem[]` |
| GET | `/inventory` | `{groups,hosts}` |
| GET | `/secrets` | `SecretRef[]` (references only — never values) |

Response shapes match `web-ui/src/data/types.ts` so the UI binds with minimal change.

## Runner

For each job run: fetch SSH private key from Vault → temp file (0600); generate an
inventory mapping the job's `limit` group → the bundled `target` host; run
`ansible-playbook <playbook> -i <inv> --limit <limit> --private-key <key>`;
capture exit/duration/stdout. Record a `Run`; push metrics to Pushgateway
(`ansible_job_last_success|_exit_code|_duration_seconds|_last_run_timestamp_seconds{task=...}`)
and the log tail to Loki (`{job="ansible-cron",task=...,status=...}`).

## Seed repo (`rudder/fleet` in Gitea)

```
ansible/jobs.yml          # manifest: ping-fleet, set-motd (dsc), disk-check
ansible/playbooks/*.yml   # real playbooks run against the target
rudder.yml                # operational config (reconcile interval, vault, alerts)
```
Connection/auth (inventory + SSH key) is supplied by the control-plane from Vault,
never stored in the repo.

## Scope / non-goals (this phase)

In: reconcile loop, cron scheduler, run-now, real Ansible over SSH, Vault SSH key
+ secret references, metrics + logs, REST API, UI live binding, GitHub/ADO
adapters. Out: PR-based editing (UI stays read-only), RBAC/multi-user, run
concurrency limits, multi-environment, durable run history across restarts
(in-memory + Loki/Prometheus are the durable sinks).

## Testing (definition of done)

`docker compose up` brings the whole stack healthy → Gitea seeded → control-plane
clones + renders jobs → scheduler fires (and run-now works) → real Ansible runs
against the target succeed → metrics in Prometheus, logs in Loki → web-UI (live)
shows real jobs/runs/logs/inventory and "add repo" works through the API.
